# Recipe: Teacher observation feedback

Goal: turn Shafaq's rough observation notes into a warm, professional, **evidence-based**
feedback draft the teacher can actually act on. This is a **draft** — Ellie never sends it
anywhere. Save it as a file and let Shafaq review/send.

## 1. Check I have what I need
I need, at minimum:
- **Who** — teacher name (or "the teacher"), subject, grade/level.
- **What was observed** — Shafaq's notes: what happened, specific moments, any concerns.

If the "what was observed" notes are missing, do NOT invent a lesson. Mark the job
**needs a short question** and ask: *"What did you observe — a few notes on what happened
in the lesson?"* Do not guess.

Optional (use if given): the observation framework/rubric being used, the tone Shafaq wants,
and any prior feedback to build on.

## 2. Ground it in good practice
Follow the sourced tips in `outputs/observation-feedback-tips.md` if present:
- Anchor every point in **specific observed evidence** (a real moment), not vague praise.
- Lead with a **genuine strength**.
- Offer **one clear, actionable next step** (as an option to try), not a long checklist.

## 3. Write the draft in this shape
```
Feedback for: <teacher, subject, grade>   ·   Observation date: <date if known>

Warm opener (one sentence, specific to the lesson).

✅ Strengths (2–3, each tied to a moment you observed)
  - <what they did> → <the effect on students>

🌱 Area to grow (1, sometimes 2 — never a laundry list)
  - <observed moment> → <why it matters> → <a concrete next step to try>

Encouraging close (one or two sentences). Offer to follow up.
```
Keep it short, professional, kind, and honest. Match Shafaq's style: short and friendly.
Never fabricate student names, scores, or events that aren't in the notes.

## 4. Save it
Write the draft to `outputs/feedback-<teacher-or-topic>-<short-date>.md`.

## 5. Report back
Return: what you drafted, the file path, and a note that it's a **draft for Shafaq to review
and send** (Ellie does not send it). If anything was assumed, say so plainly.

## Verifier will check
Completeness (covers strengths + growth + next step), Accuracy (only uses what was in the
notes — nothing invented), Usability (ready to review and send). Must score 4–5, nothing below 3.
